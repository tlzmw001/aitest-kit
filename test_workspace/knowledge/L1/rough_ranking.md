# 粗排

在精排（打分）之前对候选券进行筛选和排序，由实验开关控制是否启用。

## 输入

- 候选券列表（items）
- 实验参数（控制粗排行为）

## 输出

经过筛选和排序的候选券子集，进入特征抽取和打分阶段。

## 业务规则

### 基础能力

1. 实验开启 → 按规则截断候选券；实验关闭 → 跳过粗排
2. 原有截断方式：top_value / top_min_spend / random

### 增强能力（均通过实验参数控制，不配置则不生效）

3. **优先券保送**：isPrior=true 的优惠券优先进入精排
   - 可配置优先券入选数量上限（优先截断数）
   - 可配置优先券挑选方式：按面额高低 或 随机
   - 优先券不够时有多少选多少
   - 选完优先券后，剩余名额留给普通券

4. **条件过滤**：在粗排阶段过滤不符合条件的券
   - 支持按字段设门槛，如：过期天数 >= 3天、面额 >= 5元
   - 支持按类型筛选，如：只保留折扣券和现金券
   - 多条件同时生效，全部满足才保留

5. **多维度排序**：综合多个因素排名
   - 可配置多个排序因素及权重
   - 正数权重 = 越大越好，负数权重 = 越小越好
   - 例：面额高优先，门槛太高降权

6. **类型打散**：控制同类型券的入选上限
   - 按优惠券类型分组，配置每组最多入选几个
   - 优先保证高分券入选，兼顾类型多样性
   - 多样性是软约束，截断数是硬目标：不够时按分数补满

7. 向后兼容：以上增强能力全部通过实验参数控制，不配置则不生效

## 错误场景

- 候选券为空 → 直接返回空列表，不报错，pipeline 继续（特征抽取、精排拿到空列表，最终返回空推荐结果）
- 实验参数配置异常 → 不报错、不跳过粗排，逐项降级到默认值：
  - truncate_count 非数字 → 默认不截断（用候选券总数）
  - truncate_count ≤ 0 → 返回空列表
  - truncate_rule / prior_rule 未知值 → warning 日志 + fallback 到 top_value
  - sort_keys 中元素格式异常（非 dict / field 非 str）→ 跳过该 key
  - sort_keys 中 weight 非数字 → 按 0.0 处理
  - filters 中条件非 dict / 操作符未知 → warning 日志 + 该条件不通过
  - diversity 未启用或格式异常 / max_per_group 非数字或 ≤ 0 → 跳过打散，直接截断
  - 整体设计：每个阶段独立容错，参数异常时该阶段降级为"不生效"，不抛异常中断 pipeline
