# 0405 — 校准增强与 AB 实验服务拆分

## 变更概述

校准模块大幅增强（条件匹配、分段函数、版本化文件）；AB 实验从 SDK 进一步拆分为独立服务。

## 影响的 L1 模块

- 校准 — 新增条件匹配、分段函数校准、版本化校准文件
- AB 实验分流 — 从本地 SDK 改为调用远程服务
- AB 实验服务 — 新模块，独立部署

## 新增/变更规则

### 一期：校准增强

1. 根据 scene_id 选取校准实验
2. 校准实验含开关和校准文件目录路径
3. 校准文件包含命中条件（由请求字段组成），靠上优先级高
4. 校准文件目录下选取序号最大的文件（最新版本）
5. 两类校准：线性（y=k*x+b）和分段函数（不同区间不同 k、b）
6. 两类文件放不同目录
7. 命中逻辑：先看开关 → 条件判断命中哪类 → 都命中则先分段后线性

### 二期：AB 服务拆分

8. AB 实验拆分为独立服务，HTTP 接口，默认端口 8100
9. SDK 分远程模式（RemoteABExperimentSDK）和本地模式（ConfigBasedABExperimentSDK）
10. 服务支持实验 CRUD 和白名单管理
11. 增删改自动持久化
12. 白名单持久化到本地文件，重启自动恢复
13. 未来规划：前端平台管理实验和白名单 [?当前未实现]

## 影响分析

- 校准模块复杂度大幅增加，原有简单 k/b 逻辑被扩展
- AB 实验从进程内调用改为 HTTP 远程调用，新增网络依赖
- 主服务启动前需先启动 AB 实验服务
