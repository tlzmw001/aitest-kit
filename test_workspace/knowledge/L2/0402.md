# 0402 — 外部打分、日志系统与请求参数调整

## 变更概述

新增外部打分服务路由、请求日志系统，多个参数从配置改为请求传入。

## 影响的 L1 模块

- 参数校验与限流 — 新增 external、reqId、max_claim_per_request、score_threshold 请求字段
- 场景路由 — 兜底分数获取改为先读 Redis
- 特征抽取与打分 — 新增外部打分服务（HTTP，base_score=0.2）；内部 base_score 降为 0.1；外部请求加密 user_id
- 发放 — score_threshold、max_claim_per_request 从配置改为请求传参
- 日志 — 新增请求级日志系统

## 新增/变更规则

1. 内部打分 mock 服务 base_score 从原值降为 0.1
2. `max_claim_per_request` 和 `score_threshold` 从配置文件改为请求传参，业务方可控
3. 兜底策略：先读 Redis 获取兜底分，读不到再取配置中的固定值
4. 新增 `external` 字段（int）：0=内部打分（gRPC），1=外部打分（HTTP）
5. 外部打分服务 base_score=0.2，仅支持 HTTP，不支持 gRPC
6. 请求外部服务时，对 user_id 加密，不暴露原始 ID
7. 新增 `reqId` 字段（字符串），业务方填写用于 debug
8. 新增日志：每次请求打印 INFO 日志，含 reqId、user_id、item ID、route、scene_id
9. `route`：1=内部，2=外部，仅用于日志，不下发给打分服务
10. 场景与 route 路由隔离：external=1 时也正常计算场景

## 影响分析

- 原有 /api/v1/recommend 接口请求结构变化（新增字段）
- 原有从配置读取 threshold/claim 数量的逻辑被移除
- 原有兜底固定返回 0.5 的逻辑被替换为 Redis 优先
