# 0404 — AB SDK 化、场景实验映射与粗排增强

## 变更概述

AB 实验模块从业务代码拆分为独立 SDK；新增场景到实验的映射关系；粗排新增四项高级能力。

## 影响的 L1 模块

- AB 实验分流 — 从直接读取配置改为通过 SDK 调用；新增场景-实验映射；外部打分不获取实验
- 粗排 — 新增优先券保送、条件过滤、多维度排序、类型打散

## 新增/变更规则

### 一期：AB SDK 化

1. AB 实验模块转为 SDK 形式，作为外部依赖引入
2. SDK 新增白名单功能，可自定义 user_id 强制命中指定策略

### 二期：场景实验映射

3. 新增场景 ID → 实验列表的映射配置
4. 确定场景 ID 后才确定要读取哪些实验
5. 使用外部打分系统（external=1）时，不获取任何实验

### 三期：粗排增强

6. **优先券保送**：isPrior=true 的券优先入选；可配优先截断数和挑选方式（按面额/随机）
7. **条件过滤**：按字段设门槛（如过期天数>=3、面额>=5）；按类型筛选；多条件 AND
8. **多维度排序**：多排序因素+权重；正权重越大越好，负权重越小越好
9. **类型打散**：按类型分组配上限；多样性是软约束，截断数是硬目标
10. 以上能力全部通过实验参数控制，不配置则不生效，向后兼容

## 影响分析

- 原有直接读 experiments.json 的逻辑被 SDK 替代
- 原有全量评估实验改为按场景过滤
- 粗排能力扩展但向后兼容
