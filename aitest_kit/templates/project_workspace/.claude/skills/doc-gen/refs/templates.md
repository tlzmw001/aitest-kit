# doc-gen 输出模板参考

## _overview.md

```markdown
# 系统设计概述

## 系统功能
（一段话描述系统做什么）

## 服务拓扑
（服务间关系、协议、端口）

## 技术栈
（语言、框架、存储、协议）

## API 列表
| 接口 | 协议 | 路径/方法 | 说明 |

## Pipeline 概览
（处理流程的有序步骤，每步一句话）

## 模块列表
| 模块 | 职责 | 对应源文件 | 详细设计 |

## 配置结构
（主配置文件的完整字段说明：名称、类型、默认值、取值范围）
```

行数限制：≤ 150 行。

## module_\<name\>.md

```markdown
# <模块名>
<一句话功能描述>

## API 契约

### 输入
| 字段 | 类型 | 必填 | 默认值 | 取值范围 | 说明 |

### 输出
| 字段 | 类型 | 说明 |

## 业务规则
1. [代码] 当 xxx 时，行为是 yyy
2. [文档] ...
3. [差异] 文档说 xxx，代码实际 yyy

## 错误处理
| 触发条件 | 错误码/行为 | 是否降级 | 来源 |

## 依赖与外部调用
- 依赖名 → 协议 → 超时配置 → 不可用时行为

## 数据存储
- Redis Key 格式、Value 类型、TTL
- 文件格式、字段说明

## 可观测性
- 日志：级别 + 内容 + 触发条件
- 盲区：关键路径缺日志/监控的地方
- 已定义未使用：错误码/常量

## 实现参考
> 本节仅供开发者定位代码，不作为测试依据。

| 行为 | 源文件 | 关键函数 |
```

行数限制：≤ 300 行。

## _data_flow.md

```markdown
# 端到端数据流

## 主流程
（按步骤编号，每步说明输入→处理→输出）

## 异常分支
（每个异常路径：触发条件 → 降级行为 → 最终响应）

## 数据结构变化
（关键数据在 pipeline 中的字段变化，如字段新增/丢失/重命名）
```

行数限制：≤ 200 行。

## _discrepancies.md

仅当 `$doc_dir` 非空时生成。

```markdown
# 代码与文档差异报告

## 文档有但代码未实现
| 功能 | 文档出处 | 说明 |

## 代码有但文档未覆盖
| 行为 | 所属模块 | 说明 |

## 描述不一致
| 主题 | 文档描述 | 代码实际行为 | 影响 |

## 已定义但未使用
| 类型 | 名称 | 定义位置 | 说明 |
```

行数限制：≤ 200 行。
