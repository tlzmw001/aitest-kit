# Mismatch 记录格式

> 本文档定义规格与实现不一致时的记录格式。
> 引用方：test-design skill（第二轮发现 mismatch 时记录）、test-fix skill（修正用例时参考）

## 什么时候产生 mismatch

test-design 第二轮（读代码）发现知识库描述与代码实际行为不一致时，**不修改第一轮用例**，新建 mismatch 记录。

## 记录格式

```markdown
### MISMATCH-{序号}：{一句话描述}
- **关联**：L1/xxx 或 L2/xxxx
- **知识库描述**：知识库中的规则原文
- **实际实现**：代码中的实际行为
- **影响**：对用户/业务的影响判断
- **建议**：修代码 / 补文档 / 待产品确认
```

## 存放位置

- 有 mismatch 时输出到对应 suite 目录下的 `mismatch.md`
- 无 mismatch 则不创建该文件
- 序号在模块内连续递增，从 001 开始

## 处理流程

mismatch 记录产生后，由用户决定后续动作：
1. **修代码** — 代码行为不符合设计意图，提交 bug fix
2. **补文档** — 代码行为正确但文档未更新，刷新知识库
3. **待产品确认** — 设计意图不明确，需要产品团队澄清
