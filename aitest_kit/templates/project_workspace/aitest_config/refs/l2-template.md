# L2 需求变更文档模板

> 本模板定义知识库 L2（需求变更文档）的标准结构。
> 引用方：knowledge-build skill（产出 L2 文档）、test-design skill（读取 L2 生成增量用例）

## 模板

```markdown
## 变更概述
一句话描述

## 影响的 L1 模块
- 模块名 — 具体影响点

## 新增/变更规则
1. ...

## 影响分析
- 哪些已有功能可能被影响，需要回归

## 测试重点
- 新逻辑的关键路径
- 容易出问题的边界
```

## 约束

- 每个文件控制在 100 行以内
- 描述变更 delta，不重复 L1 已有内容
- 测试重点要写到可以直接映射为用例的粒度
